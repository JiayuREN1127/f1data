# 大作业要求

本次大作业旨在综合运用课程所学方法，完成一个相对完整的小型研究设计与实证分析项目。

请围绕一个自拟的研究问题，完成从理论构建、数据获取或生成、变量测量到统计分析的全过程，并在课堂进行汇报。

首先，汇报不要求进行理论分析、文献综述等，可直接提出hypothesis。在研究设计方面，需提出具有一定理论意义且具备复杂性的研究问题，并至少提出三个及以上明确假设。

鼓励采用相对复杂的模型形式，例如

- 调节的中介模型（moderated mediation）
- 跨层作用（cross-level effect）
- 时间滞后/变化模型（如growth或lagged effects）

在数据方面，

- 可使用已有科研数据，或使用GPT生成模拟数据
- 如使用模拟数据，必须体现清晰的嵌套结构（multilevel），例如员工嵌套于团队、或时间嵌套于个体的日记数据结构，并确保数据结构与研究问题相匹配

在测量方面，

- 每个核心变量需提供量表测量，每个变量至少包含三个及以上条目
- 量表条目可基于已有文献改编，也可由GPT生成，但需具备合理性
- 同时，需要完成基本的信度分析（如Cronbach’s α）和因子分析（Factor Analysis），并在适用情况下报告聚合指标（如ICC(1)、ICC(2)、rwg等）

在统计分析方面，

- 必须体现多层线性模型（MLM）与因子分析两类方法的应用
- MLM部分至少包含一个multilevel model，并可根据研究需要设置随机截距或随机斜率，以及跨层交互效应
- 因子分析部分可采用EFA或CFA，但需对因子结构进行合理解释

在结果呈现方面，本课程强调真实的研究操作过程，而非论文式美化呈现。

- 要求在PPT中直接粘贴统计软件的命令（如Stata、Mplus或R）以及原始输出结果截图，不需要整理
- 重点在于展示实际分析过程，并体现对结果的理解，而非形式上的规范化

本次作业需提交完整的PPT文件，并在课堂进行口头汇报

每位同学或每组汇报时间建议为15–20分钟（可根据实际人数适当调整）

评价将主要基于以下几个方面：

- 研究问题是否有趣且具有逻辑性
- 模型设定是否合理（尤其是multilevel结构）
- 方法使用是否规范（MLM与因子分析）
- 是否真正理解并解释分析结果
- 以及整体汇报表达的清晰度

最后说明，本作业不要求“完美研究”，或必须要发现显著性结果，但需做到清晰、规范且可解释。可以在模型上适当增加复杂性，但应确保理论与方法逻辑一致。如使用GPT生成数据，需特别注意数据结构的合理性及其与理论设定的一致性。
