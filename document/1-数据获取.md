# F1 数据采集与变量生成说明

**文件说明**：本文档记录从 FastF1 API 爬取 F1 赛事原始数据、检测并清洗超车事件、生成分析变量的完整过程。复现框架来自 Hallila, Frankort & Aversa (2024)，发表于 *Academy of Management Journal*（以下简称"原文"）。

---

## § 0 项目概述

### 0.1 研究背景

原文以 MotoGP 2004–2020 赛季（rider-lap 级面板）为实证场景，检验车队层级的竞争压力（competitive threats/opportunities）如何对车手的内部/外部竞争行为（超车）产生跨层效应，以及相对资源禀赋的调节作用。本项目将该框架迁移至 **F1（一级方程式赛车）2018–2026 赛季**，以 FastF1 提供的官方遥测数据为基础，构建类比变量。

## § 1 API 爬取与缓存

### 1.1 FastF1 库与缓存机制

FastF1 是 F1 官方授权的 Python 数据接口库，通过本地文件缓存避免重复下载：

```python
fastf1.Cache.enable_cache("./cache")   # 首次下载后缓存至本地
```

缓存目录结构为 `cache/{year}/`，同一 session 只需下载一次，后续可离线运行。**每场比赛的遥测数据约 50–200 MB，首次全量爬取需数小时，请确保磁盘空间充足（建议预留 ≥ 20 GB）。**

### 1.2 核心 API 对象

```python
# 获取赛历（返回 DataFrame，含 RoundNumber, EventName, Location 等）
schedule = fastf1.get_event_schedule(year, include_testing=False)

# 获取特定场次的 Session 对象
sess = fastf1.get_session(year, round_number, session_type)
# session_type: 'R'（正赛）/ 'Q'（排位赛）/ 'SQ'（冲刺排位）

# 按需加载数据（减少不必要的 API 请求）
sess.load(laps=True, telemetry=False, weather=True, messages=False)
```

| 参数 | 含义 | 请求代价 |
|------|------|------|
| `laps=True` | 加载逐圈数据（位置、圈时、状态等） | 中等 |
| `telemetry=True` | 加载车辆遥测（速度/油门/刹车/DRS，约 10 Hz） | 高（最大） |
| `weather=True` | 加载天气时序数据 | 低 |
| `messages=True` | 加载赛事消息（用于状态解析） | 低 |

### 1.4 `1-overtakes.ipynb`：超车采集流程

每站正赛分**两阶段** load，避免一次性加载所有遥测数据消耗过多内存：

```
阶段 1: R session.load(laps=True, messages=True)
    ├── 构建 laps_base（逐圈位置、Pit、TrackStatus）
    └── 检测超车候选（位置差分，无需遥测）

阶段 2: R session.load(telemetry=True)
    └── 仅对候选超车涉及的车手 × 圈次按需拉取遥测
        → 建立 tel_store 缓存，计算 10 维特征（D1–D10）
```

Cell H 通过参数控制采集范围：

```python
ALL_YEARS  = list(range(2023, 2026))   # 目标赛季
SKIP_YEARS = {2024}                    # 已有数据，跳过
```

每站处理完毕后**立即存盘**（防止中途崩溃丢失数据）。

## § 2 清洗与变量生成

### 2.1 超车候选事件检测（位置差分法）

核心思路：**比较相邻两圈的赛道位置**，若某车手 A 在第 L-1 圈排名落后于 B、但在第 L 圈排名领先于 B，则记录一次"超车候选"。

具体步骤：

1. **构建位置矩阵**：对全场 `laps_base` 做 `pivot_table(index='LapNumber', columns='Driver', values='Position')`，得到每圈每车手的位置矩阵 `lep`（lap-end position）
2. **计算前一圈位置**：`lsp = lep.shift(1)` 得到 lap-start position
3. **枚举超车对**：遍历所有圈次，对两两车手 (A, B) 判断是否发生位置倒转：
   - 严格条件：`pa_s > pb_s`（A 圈初落后于 B）且 `pa_e < pb_e`（A 圈末领先于 B）
4. **附加元信息**：`LapStartTime`、`SessionTime`（圈结束时刻）、`IsTeammate`（同队标记）

> **位置数值方向**：FastF1 `Position` 为"赛道名次"，**数值越小越好**（1st = 1，2nd = 2）。"A 超越 B"对应 `pa_e < pb_e` 且 `pa_s > pb_s`（A 从落后到领先）。

### 2.2 Pit/DNF 影响剔除

位置变化可能由进出维修站或车手退出造成，而非真实超车：

| 情形 | 剔除逻辑 |
|------|----------|
| **A 刚从维修站出来**（Pit Out） | 若 A 在本圈或上一圈有 `PitOutTime`，排除该超车候选 |
| **B 进入维修站**（Pit In） | 若 B 在本圈或上一圈有 `PitInTime`，排除该超车候选 |
| **B 已退赛**（DNF） | 若 B 在 `sess.results` 中 `Status` 不含 "finished"/"laps"，且其最后记录圈 `Position` 为 NaN，则视为退赛，排除 |

上述剔除后，`PitInfluenceFlag` 和 `DNFInfluenceFlag` 字段在当前实现中均为 `False`（预留字段，表示已通过模块化逻辑过滤，而非逐条标记）。

### 2.3 10 维遥测特征批量计算

对每条超车候选（超车方 A，被超车方 B，圈次 L），从 `tel_store[(driver, lap)]` 中提取遥测计算（详见 § 5.4）：

| 特征编号 | 字段名 | 简述 |
|---------|--------|------|
| D1 | `Is_DRS_Active` | A 在该圈是否激活 DRS（DRS 信号值 = 10） |
| D2 | `RelativeDistance` | A 该圈遥测 `RelativeDistance` 的中位数（赛道位置比例） |
| D3 | `Throttle_Drop_Index` | B 在加速段（40–80% 赛道）相对 A 的收油幅度（越大越可疑为主动被超） |
| D4 | `Brake_Pressure_Delta` | 两车刹车上升沿时序差中位数（毫秒，越大越可疑） |
| D5 | `TeamOrderConfidence` | 车队指令置信度（由 D3、D4 加权合成，0–1） |
| D6 | `Vmin_Deviation` | B 在关键刹车区最低速度相对参考圈的偏差百分比 |
| D7 | `OffTrack_Flag` | B 在该圈遥测中 `Status='OffTrack'` 占比 > 5% |
| D8 | `Lockup_Flag` | B 出现刹车同时油门 > 30 的异常组合（≥ 2 帧） |
| D9 | `LocalYellowFlag` | 该圈 `TrackStatus` 包含黄旗代码 `'2'` |
| D10 | `OffTrackConfidence` | 越界/锁死置信度（由 D6–D9 加权合成，0–1） |

### 2.4 车队指令超车过滤

满足以下两个条件的超车候选被标记为**疑似车队指令**，存入 `team_order_season`：

```
IsTeammate == True
AND TeamOrderConfidence ≥ TEAM_ORDER_CONFIDENCE_THRESHOLD（默认 0.5）
```

**阈值依据**：`TEAM_ORDER_THROTTLE_THRESHOLD = 10.0`（油门差百分比触发值）、`TEAM_ORDER_BRAKE_THRESHOLD = 150.0`（刹车时序差毫秒触发值），两者加权合成最终置信度（权重 0.6 + 0.4）。

### 2.5 越界/锁死超车过滤

满足以下两个条件的超车候选被标记为**越界/锁死/黄旗相关**，存入 `off_track_season`：

```
(OffTrack_Flag == True OR Lockup_Flag == True OR LocalYellowFlag == True)
AND OffTrackConfidence ≥ OFF_TRACK_CONFIDENCE_THRESHOLD（默认 0.5）
```

**权重构成**：`Vmin_Deviation` 超阈值（+0.4）、`OffTrack_Flag`（+0.3）、`Lockup_Flag`（+0.2）、`LocalYellowFlag`（+0.1）。

### 2.6 净超车（clean_overtakes）派生

```python
# key = (Round, LapNumber, Overtaker, Overtaken)
dirty = set(zip(team_order.Round, team_order.LapNumber, team_order.Overtaker, team_order.Overtaken)) \
      | set(zip(off_track.Round,  off_track.LapNumber,  off_track.Overtaker,  off_track.Overtaken))

clean_overtakes = full_overtakes[~full_overtakes.key.isin(dirty)]
```

即从全量超车中**去掉**属于 `team_order` 或 `off_track` 的记录，得到净超车。

## § 3 核心变量与文献对应

### 3.1 F1 vs MotoGP 调适说明

原文使用 MotoGP 数据，本项目迁移至 F1，两者在数据结构和变量操作化上存在以下主要差异：

| 维度 | Hallila et al. (2024) MotoGP | 本项目 F1 |
|------|------------------------------|----------|
| **数据来源** | motogp.com 官方逐圈扇区数据 | FastF1 官方遥测（精度更高，含车辆物理数据） |
| **分析单元** | rider × lap（逐圈观测，104,606 条） | driver × race（逐站观测，聚合超车次数） |
| **因变量测量** | 原始逐圈超车计数 | 每站净超车次数（经 3 步清洗后基于圈级事件聚合） |
| **资源禀赋操作化** | factory team vs satellite team 的二元区分（公认层级划分） | `quali_gap`：本站排位赛车队最快圈时 − 杆位时间（秒），连续变量 |
| **竞争威胁/机会** | 逐圈计算 striking distance，赛中实时更新 | 以赛前（赛季累积积分）为基准，每站全场统一计算 CT/CO *(待 `2-data.ipynb` 实现)* |
| **超车数据质量控制** | 基于官方数据库，含 missing data 处理 | 增加遥测特征（D1–D10）主动清洗车队指令/越界干扰 |
| **赛制差异** | MotoGP 每站 1 场比赛 | F1 含冲刺赛周，标准化为主赛 R session + 主排位 Q session |

**资源禀赋说明**：MotoGP 中 factory/satellite 是公认的资源分层（整车制造能力、年度预算约 5,000 万 vs 1,500 万美元等）。F1 的对应操作化方式采用**排位赛圈时差**（`quali_gap_seconds = team_best_Q_time − pole_time`），体现单站赛前各车队相对于最强车队的技术差距，是当赛季该站赛道上资源禀赋的连续化直接反映。值越大，资源越处于下风。

## § 5 超车及复杂变量计算逻辑

### 5.1 位置矩阵与超车配对

```python
# 1. 构建圈末位置矩阵（行=圈次，列=车手，值=赛道名次，越小越靠前）
lep = laps_base.pivot_table(
    index='LapNumber', columns='Driver', values='Position', aggfunc='first'
).sort_index()

# 2. 计算圈初位置（即上一圈末的位置）
lsp = lep.shift(1)

# 3. 遍历每圈，枚举所有 (A, B) 配对
for lap_n in sorted(lep.index):
    if lap_n == 1: continue     # 第 1 圈无前序，跳过
    ps = lsp.loc[lap_n].dropna()
    pe = lep.loc[lap_n].dropna()
    common = ps.index.intersection(pe.index)
    for a in common:
        pa_s, pa_e = ps[a], pe[a]
        if pa_e >= pa_s: continue  # A 名次没有提升，排除
        if a in pit_out_set: continue  # A 是从维修站出来的，排除
        for b in common:
            if b == a: continue
            pb_s, pb_e = ps[b], pe[b]
            # 超车判定：A 圈初落后于 B，圈末领先于 B
            if pa_s > pb_s and pa_e < pb_e:
                if b not in pit_in_set and b not in dnf_set:
                    record(a, b, lap_n, ...)
```

### 5.2 Pit/DNF 剔除细节

```python
def _pits(lap_n):
    """返回该圈有 PitIn 和 PitOut 记录的车手集合。"""
    rows = laps_base[laps_base['LapNumber'] == lap_n]
    return (
        set(rows.loc[rows['PitInTime'].notna(),  'Driver']),   # pit_in
        set(rows.loc[rows['PitOutTime'].notna(), 'Driver']),   # pit_out
    )

# 覆盖当前圈及上一圈（处理跨圈的维修站进出场景）
pi, po   = _pits(lap_n)
pip, pop = _pits(lap_n - 1)

# DNF 车手集合（Status 不含 "finished"/"laps" 的车手 且 最后圈 Position 为 NaN）
dnf_drv = dnf_results & set(last_position_is_nan)

# 排除逻辑：
# A in po or pop → A 刚从维修站出来
# b in pi or pip → B 进入维修站
# b in dnf_drv   → B 已退赛
```

### 5.3 "Striking Distance" 竞争威胁/机会公式（待 `2-data.ipynb` 实现）

原文（Hallila et al., 2024, p. 1340）核心公式：

设第 r 站开赛前，焦点车队 i 的累积积分为 $P_{i,r-1}$，对手车队 j 的累积积分为 $P_{j,r-1}$。定义当站的"striking distance"：

$$S_{j \to i,\, r} = \bigl(P_{j,r-1} + \text{MaxCollectable}_{j,r}\bigr) - \bigl(P_{i,r-1} + \text{MinCollectable}_{i,r}\bigr)$$

$$S_{i \to j,\, r} = \bigl(P_{i,r-1} + \text{MaxCollectable}_{i,r}\bigr) - \bigl(P_{j,r-1} + \text{MinCollectable}_{j,r}\bigr)$$

- $S_{j \to i} > 0$：**j 有可能超越 i** → j 对 i 构成竞争威胁
- $S_{i \to j} > 0$：**i 有可能超越 j** → i 对 j 拥有竞争机会

$$\text{CT}_i = \sum_{j \neq i} \mathbf{1}\bigl[S_{j \to i,r} > 0\bigr], \quad \text{CO}_i = \sum_{j \neq i} \mathbf{1}\bigl[S_{i \to j,r} > 0\bigr]$$

**F1 实现要点**：

- **积分制**：F1 前 10 名积分为 25-18-15-12-10-8-6-4-2-1，最多两名车手参赛（最大 43 分/站）
- `MaxCollectable`：假设该队两名车手分获第 1、2 名，即 25 + 18 = 43 分
- `MinCollectable`：假设该队两名车手均完赛但排名积分区外（0 分），即 0 分
- 计算基准：以赛前（上站结束后）累积积分为准，每站更新一次（对应原文 Appendix D 的 race-to-race 版本，简化自原文的逐圈更新）

### 5.4 D1–D10 特征详细说明

#### D1 — `Is_DRS_Active`（DRS 激活）

- 取超车方 A 该圈遥测 `DRS` 信号，任意时刻 `DRS == 10` 即视为本圈 DRS 激活
- DRS 信号值含义：`0`=不可用，`8`=可用（已关闭），`10`=DRS 开启
- 用途：区分 DRS 助力超车与纯实力超车

#### D2 — `RelativeDistance`（相对赛道位置）

- A 该圈遥测的 `RelativeDistance` 列中位数（经 `add_relative_distance()` 计算）
- 值域 0–1，表示车手在赛道上的相对位置占比
- 用途：辅助判断超车发生在赛道哪个区域

#### D3 — `Throttle_Drop_Index`（油门下降指数）

- 将 A 和 B 的遥测按 `SessionTime` 做 `merge_asof`（最近邻时间匹配）对齐
- 取 `RelativeDistance ∈ [0.4, 0.8]` 的加速段（弯后直道段）
- 计算 `|median(Throttle_A − Throttle_B)|`（单位：%，0–100）
- 含义：B 相对 A 的主动收油行为强度；值越大，B 主动减速让行的嫌疑越大

#### D4 — `Brake_Pressure_Delta`（刹车时序差）

- 提取 A、B 遥测中刹车信号（`Brake`）的 0→1 上升沿时刻列表（即每次踩刹车的时刻）
- 对 A 的每个刹车事件，找 B 中时间最近的刹车事件，若时差 < 500 ms 则计入
- 取所有匹配对的时间差绝对值中位数（ms）
- 含义：两车刹车时序差越大，越可能有一方主动不刹（被超意愿）

#### D5 — `TeamOrderConfidence`（车队指令置信度）

$$\text{TeamOrderConfidence} = \min\!\left(\frac{\text{TDI}}{10.0},\,1\right) \times 0.6 + \min\!\left(\frac{\text{BPD}}{150.0},\,1\right) \times 0.4$$

其中 TDI = `Throttle_Drop_Index`（%），BPD = `Brake_Pressure_Delta`（ms）

#### D6 — `Vmin_Deviation`（最低速度偏差）

- 仅在该圈 `TrackStatus` 含黄旗（`'2'`）时有效（否则返回 NaN）
- 取 B 在 `RelativeDistance > 0.7` 段（赛道后段关键刹车区）的最低速度，与同车手历史正常圈（`TrackStatus=='1'` 且 `IsAccurate==True`）的参考最低速度比较
- `Vmin_Deviation = (current_min − ref_min) / ref_min × 100`（%；负值表示当圈比正常圈更慢）
- 用途：识别因越界/事故导致的非正常慢车被超

#### D7 — `OffTrack_Flag`（越界标志）

- B 该圈遥测 `Status == 'OffTrack'` 占所有遥测帧的比例 > 5%

#### D8 — `Lockup_Flag`（轮胎锁死标志）

- B 该圈遥测中同时满足 `Brake == 1`（刹车）且 `Throttle > 30`（带油）的帧数 ≥ 2
- 含义：刹车时意外带油门，通常表示轮胎锁死或驾驶失误

#### D9 — `LocalYellowFlag`（局部黄旗）

- 该圈 `laps_base` 中任意车手的 `TrackStatus` 包含 `'2'`（黄旗代码）
- 注：此为全场黄旗判断，非局部扇区级判断（FastF1 的 TrackStatus 为全场状态）

#### D10 — `OffTrackConfidence`（越界置信度）

$$\text{OffTrackConfidence} = \underbrace{0.4 \cdot \mathbf{1}[\text{Vmin\_Dev} > 5\%]}_{\text{极速显著降低}} + \underbrace{0.3 \cdot \text{OffTrack\_Flag}}_{\text{出赛道}} + \underbrace{0.2 \cdot \text{Lockup\_Flag}}_{\text{锁死}} + \underbrace{0.1 \cdot \text{LocalYellowFlag}}_{\text{黄旗下}}$$
